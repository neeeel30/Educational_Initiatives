# Question1

This folder contains solutions related to various design patterns. The design patterns are categorized into three main types, and each category has its own folder. For a detailed explanation and description of each design pattern, refer to the respective README files inside each folder.

## Folder Structure

### 1. [Behavioural Design Pattern](./behavioural_design_pattern)
This folder includes examples and solutions related to **Behavioural Design Patterns**. These patterns focus on communication between objects. For a detailed description, please refer to the [README](./behavioural_design_pattern/README.md) inside the folder.

### 2. [Creational Design Pattern](./creational_design_pattern)
This folder includes examples and solutions related to **Creational Design Patterns**. These patterns deal with object creation mechanisms, trying to create objects in a manner suitable to the situation. For more information, please refer to the [README](./creational_design_pattern/README.md) in this folder.

### 3. [Structural Design Pattern](./structural_design_pattern)
This folder includes examples and solutions related to **Structural Design Patterns**. These patterns focus on how classes and objects are composed to form larger structures. For further details, refer to the [README](./structural_design_pattern/README.md) located in the folder.

---

Each subfolder provides code examples and explanations specific to that category of design patterns.
